# Esta Workforce OS Browser Extension

Privacy-safe Chromium extension foundation for website tracking.

## What It Sends

The extension sends only:

- normalized hostname, for example `github.com`
- observed timestamp
- browser key: `chrome`, `edge`, `brave`, or `opera`

It does not send full URLs, paths, query strings, fragments, page content, cookies, browser history, typed text, form values, clipboard data, or screenshots.

## Build

```powershell
cd H:\esta-workforce-os\browser-extension
npm run typecheck
npm run test
npm run build
```

The built extension is emitted to:

```text
browser-extension/dist
```

## Pair With Desktop Agent

1. Start the Esta Workforce OS desktop agent.
2. Sign in.
3. In the desktop renderer dev console, run:

```js
await window.esta.browserBridge.getPairingInfo()
```

4. Copy the returned `port` and `token`.
5. Open the extension options page and paste the values.
6. Click **Test Connection**.
7. The status should show **Connected**.

The token is generated by the desktop agent for the current local bridge session. It is not a backend JWT, refresh token, password, or company secret.

## Install In Chrome

1. Build the extension.
2. Open `chrome://extensions`.
3. Enable **Developer mode**.
4. Click **Load unpacked**.
5. Select `H:\esta-workforce-os\browser-extension\dist`.

## Install In Edge

1. Build the extension.
2. Open `edge://extensions`.
3. Enable **Developer mode**.
4. Click **Load unpacked**.
5. Select `H:\esta-workforce-os\browser-extension\dist`.

## Manual Test

1. Start backend.
2. Start desktop agent and sign in.
3. Pair the extension.
4. Open `github.com` in Chrome and keep Chrome foreground for 90 seconds.
5. Switch to `chatgpt.com` for 90 seconds.
6. Switch to VS Code.
7. Wait for activity upload.
8. Confirm desktop logs show `websiteRecordCount` greater than zero.
9. Confirm `GET /api/monitoring/activity?dateFrom=<today>&dateTo=<today>` contains website records.
10. Confirm `/monitoring/activity` shows Top Website.

## Security Notes

- Bridge binds only to `127.0.0.1`.
- Extension authenticates with a bearer pairing token.
- Requests are size-limited and rate-limited.
- Payloads containing `url`, `path`, `query`, `title`, `pageContent`, `cookie`, `history`, `typedText`, form values, or clipboard fields are rejected.
- Incognito is disabled by default.
- Firefox support is intentionally not included in this Chromium foundation.
